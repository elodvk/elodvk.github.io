# PurpleSec — Project Context (Memory)

This is the persistent project memory. Keep it updated whenever the architecture changes
or new features are added. Live site: https://purplesec.org/ (GitHub Pages, repo `elodvk/elodvk.github.io`).

## What this is
A cybersecurity content site by **Bilash J. Shahi** ("PurpleSec"): HTB walkthroughs, Active
Directory attack writeups, certification prep (OSCP/CPTS/PNPT/CISSP), and a security blog.
Theme/style: dark "slate" with deep-purple/purple accent, terminal/hacker aesthetic
(matrix rain, glitch text, fake terminals).

## Tech stack & build
- **Static site generator: Zensical** (NOT plain MkDocs). Config is `zensical.toml` (TOML, under `[project]`), not `mkdocs.yml`. It is Material-for-MkDocs compatible (templates extend `base.html`, uses `md-*` classes, `document$` instant-nav hook).
- Python 3. Local dev: `zensical serve`; build: `zensical build --clean` (outputs to `site/`).
- Theme fonts: text = IBM Plex Mono, code = JetBrains Mono.
- Markdown extensions: full pymdownx suite (superfences w/ mermaid, tabbed, tasklist, arithmatex, highlight, etc.), admonition, attr_list, md_in_html, footnotes, toc (permalink).
- Plugins: `macros` (module_name = `main`), `social` (cards, bg `#0a0e14`, color `#bb86fc`), `blog` (blog_dir = `blog`), `encryptcontent`, Google Analytics (`G-5S46532J99`) with feedback widget.

## Deployment (CI/CD)
`.github/workflows/docs.yml` runs on push to `main`/`master`:
1. `pip install zensical pycryptodome beautifulsoup4`
2. `zensical build --clean`
3. `python encrypt_pages.py`  (post-processes built HTML for password-protected pages)
4. `python ping_indexnow.py`  (regenerates sitemap + submits to IndexNow)
5. Upload `site/` artifact and deploy to GitHub Pages.

## Root Python scripts
- **main.py** — Zensical macros module (`define_env`). Provides two Jinja macros:
  - `get_blog_posts()` — scans `docs/blog/*.md` (skips `index.md` & `draft: true`), parses YAML frontmatter, computes summary, date (`date_obj`/`date_str`/`year`), authors (normalized to list), tags, `read_time` (words/200), `url`. Sorts newest-first. **Side effect:** generates RSS at `docs/feed.xml` and `site/feed.xml` (top 50 posts).
  - `get_walkthroughs()` — same idea but recursive over `docs/walkthroughs/`, adds `difficulty` and `os` fields. No RSS.
- **encrypt_pages.py** — For any `.md` with `password:` in frontmatter, finds its built HTML in `site/`, AES-encrypts the `<article>` inner HTML (CryptoJS-compatible: `Salted__` + MD5 KDF), replaces content with a decryptor UI (password prompt + client-side CryptoJS decrypt), and strips the TOC. Used to gate HTB walkthroughs behind the root flag.
- **ping_indexnow.py** — Regenerates `site/sitemap.xml` from all `index.html` files, then POSTs all URLs to IndexNow (`api.indexnow.org`). IndexNow key: `9d8a3cf1e97d4b0d874c98038b30f5a2` (key file at `docs/9d8a3cf1e97d4b0d874c98038b30f5a2.txt`).
- **update_social_cards.py** — One-off helper to inject `image:` into frontmatter / banner images into bodies for blog & walkthrough files.

## Content structure (`docs/`)
- `index.md` — homepage (stats, knowledge-base cards, blog feed terminal via `get_blog_posts()`, cert grid, quote carousel).
- `blog/` — blog posts (flat `.md` files). `blog/index.md` renders cards via `get_blog_posts()`, tag filter, load-more, year archive. Banner images in `blog/assets/`, hero images in `assets/images/blog/`.
- `walkthroughs/` — HTB machine writeups (flat `.md`). Nav label is **"Pwned"**. `walkthroughs/index.md` lists via `get_walkthroughs()` (shows OS icon + difficulty). Per-machine images under `walkthroughs/assets/<machine>/`. Some are password-protected (e.g. `checkpoint.md` has `password:`).
- `writeups/active_directory/` — AD technique writeups. Nav label "Knowledge Base". Subfolders: `Initial Attack Vectors/`, `Post-Compromise Enumeration/`, `Post Compromise Attacks/`, `Post Domain Compromise/`, `adcs_attacks/` (ESC1–ESC15 + Golden Certificate), `imp_attacks/`.
- `certifications/` — `oscp/`, `cpts/`, `pnpt/`, `cissp/` each with `index.md`.
- `about/index.md` (resume template), `contact.md` (Supabase contact form), `robots.txt`, `feed.xml` (generated).

## Frontmatter conventions
- Common: `title`, `description`, `date` (YYYY-MM-DD), `tags` (list), `authors` (name/title/picture/url), `image` (path relative to docs, e.g. `assets/<machine>/banner.png`).
- Walkthroughs add: `difficulty`, `os` (Windows/Linux).
- `draft: true` hides a post from listings/RSS.
- `password: "<hash>"` gates a page behind client-side AES decryption (root flag).
- `template:` selects an override (home.html / resume.html); `hide:` controls nav/toc/title/footer/path.

## Conventions & gotchas
- Edits to navigation go in `zensical.toml` `nav = [...]`.
- New blog post = drop a `.md` in `docs/blog/`; it auto-appears in listings + RSS via macros.
- New walkthrough = drop a `.md` in `docs/walkthroughs/` with `difficulty`/`os`.
- RSS/sitemap are generated at build time — don't hand-edit `feed.xml`/`sitemap.xml`.
- This is Windows/cmd dev environment; site is on GitHub Pages.
